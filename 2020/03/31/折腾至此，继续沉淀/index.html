折腾至此，继续沉淀

发表于 2020-03-31 | 分类于 博客探索 ， WordPress | |
字数统计: 943 | 阅读时长 ≈ 3

2020年的三月底，我在博客的事情上经历了较多尝试：“历久弥新”的Hexo到WordPress初探（com），到阿里云仅仅是用了半个月后的弃用，现转至现在Vultr主机。

希望不会出现什么问题，否则只会迁移主机，再不行回到之前的静态或直接Joplin——或者，Google Blogger。

经历了些折腾，当前希望自己能在这里安定下来，稳定了。

由此可见，现在的环境里讲讲话有多难，哪怕即使想放是一个社交连接的图标，一个Twitter的卡片。

新服，WordPress

“我该如何安装新服务器上的WordPress？”

最近几天，我一直在考虑这个问题。

殊不知昨天晚上在选择主机的时候因为有点强迫症，在关键的时候因为一个动作想到了与这无关的东西，于是关闭了Vultr的账号，后提交工单，账号方恢复。

今晚，我搜寻了较多的资料，发现用主机安装WP无非有以下几种（以下均包含VPS主机）：

  1. WP客户端+（CentOS）+LNMP+宝塔+Xshell/终端（各种教程或多或少均有涉及）
  2. WP客户端、Nginx、MySQL、PHP独立安装
  3. 自带WP模版一键安装

不信任国内的面板与信息安全的我，有了和产品经理的以下对话。手机在楼上充电，你知道我要说什么。

手机在楼上充电，你知道我要说什么。

无处安放的历史文章

在这里，说说我对于博客里“导入导出文章”的理解，以及内心的纠结所在。

轻量级服务器旧闻

在使用阿里云轻量级服务器的时候，我便写了两篇文章，一篇是关于搭建的，另一篇则是自己的感慨的。

现在，他们都在我的“下载”文件夹里，我不愿意在这里导入，可能和我的“喜新厌旧”的心理有关。

如果导入了，看到之后会想起国内审查这个事实，也想起我曾经在这里留下过我的记录。

不仅仅是我，阅读过我的老站文章的读者们都知道，我会将生活随笔与我的自学内容前不妨在一个博客里，HR看到后（在墙内生活工作，尤其是大厂）看到这些文章之后，怎么可以接受一个（即使是曾经）抵触自己产品的求职者呢？

老站Hexo导入

在之前的时候，我搜寻过从Hexo迁移至WordPress的几篇文章，里面说可以使用插件进行快速导入：

FeedRSS，通过获得RSS的方式进行导入。

凡事总有那么个但是，由于我做某一件事通常需要花很长的时间与精力才能做好一件事，尤其是需要类似探索与记录探索过程的时候，花的时间则会更长。

如果迁移到一半，网络突然中断（学校），或者生活中有了其他临时事情的时候，我便不得不停下这件事：

稍后再做——再也不做。

这样，老站就会成为我的半成品，是一种尴尬的存在。

现在老站我会保留，更好的是因为在本地有Git文件。

当然，WordPress也要定期导出，这篇也是。

近况、待办与后记

等待阿里云的退款，开始配置新站WordPress，继续说说心里话，表达内心的真实写照。

可能，后面的某一天，你会看到我的3月15日那天，会有一篇文章；也有可能，你会看到，我在2020年之前的部分与全部记录，这都说不准。

产品北里，讲述产品和人性、涉及设计、技术和运营的公众号。
你们的赞赏是我更新的动力～
