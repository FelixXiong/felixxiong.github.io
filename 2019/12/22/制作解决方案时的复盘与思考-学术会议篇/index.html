制作解决方案时的复盘与思考-学术会议篇

发表于 2019-12-22 | 分类于 工作 ， 行业解决方案 | |
字数统计: 3.6k | 阅读时长 ≈ 12

使用近乎三周的时间和同事协作，撰写与制作了第一个活动与会议行业的解决方案幻灯片。

现在来复盘与总结，本篇文章不仅为学术会议，并且广泛适用于解决方案与客户案例的通用幻灯片制作方法。

分享干货不易，如您愿意可扫码请我喝杯luckin，继续推动我的原创干货分享。

若看到国内外平台有抄袭与洗稿等现象，欢迎点击左侧邮箱给我发送邮件。

本次制作过程中，改动明显较多的为三大点：文案、幻灯片单页样式、图片。

本文使用自己前几天总结的经验内容的增加，发现自己写的复盘文章，结构越来越清晰，小点愈发细分了。

权利声明：本篇文章仅为个人总结经验分享，涉及公司内部相关资料，皆隐去。文末段为个人习惯性思考发问。

概览

我使用为期一周的时间和同事进行第一个版本的框架搭建，根据不同解决方案进行梳理主要流程。

协作人员：我、运营部同事及其他部门同事。

制作前参考与调研：不同行业解决方案、本公司产品解决方案

解决方案的主要流程

公司简介-解决方案简介-产品优势-应用场景-产品介绍-客户案例-合作伙伴-联系我们

框架搭建

框架搭建时，参考各种行业解决方案进行删减并确认，文案切勿过早填充，避免无用功。

在（包括但不限于）学术会议解决方案中，部分内容需进行内容考量是否添加：

  1. 例如学术会议的种类：paper、short paper等分类，及研讨会的普通研讨会以及非普通研讨会等，因为客户案例难以进行分类，则采取“若无必要，勿增实体[1]”。
  2. 申请试用页面

其他根据公司需要可进行相应增删的页面也是需要多去思考是否有必要增加。

制作过程中不断与反复进行草稿确定，同样可避免无用功。关于草稿，会在近文末段中进行提出。

文案

与同事协作，自己考虑到撰写的公平性 [2]，文案也进行分工协作，后续增加部分内容导致制作时间增加。

协作工具：Word与后语雀进行通用文案沉淀

文案考虑点：

  1. 语句通俗易懂，切勿使用专业的名词，拒绝让读者思考。
  2. 错位与手误：文案错位问题、输入过程手误（错别字）、语病、倒装
  3. 页面内容基本相同的页面删减
  4. 根据已有图标的流程定制（如有必要）与删改核实，避免出错
  5. 需考虑文案的不断更改对插图所表达含义的影响
  6. 聊天中的成段文案复制与记录
  7. 文案修改定时同步协作工具

不同区域文案注意点

标题与内容

确定标题：学术研讨会/学术会议，并统一标题。每张幻灯片导航小标题进行与导航统一，阐述部分内容。

我和同事分别撰写不同的内容，需要考虑到不同时间节点。

行业痛点

内容确定：提前咨询是否有补充，避免后续增加修改时长。

此处可以对接：市场部

产品功能

  1. 介绍内容时使用不同表达方式
  2. 统一：分页与汇总页面功能点名称、前后页面数据、页面导航名称
  3. 考虑文案因果关系，例如活动现场的流程。
    此处可以增加一些人性化
用户路径（如果有）

产品迭代：

  1. 用户路径的删减
  2. 不同同事对于入口的理解

合作伙伴

社会团体与组织机构的称呼与页面介绍先后顺序的差异

####客户案例

本case中收集10例客户案例，与Logo一并汇总整理文档留存，包含中英文案例、链接等。

收集尽可能多的客户案例，之后进行汇报删减，注意选取好看的客户案例。

来源：新闻通稿

文案介绍注意先后顺序：报名注册、现场签到、专家莅临、成果与奖项

是否使用我们的签到系统，此处对接：市场部

不用每个活动都要相应介绍，例如即将召开的数据，找准确数据难

此处对接：内容运营、市场部-Logo、设计部-设计图资源

迁移文案

在同事与领导可接受的前提下，我们使用语雀文档进行工具的沉淀，在此很高兴进行认可与同意。

关于工作中的习惯：

自己根据已有工作习惯养成后，部分文案放必要链接即可，若无特殊要求，可以不放。

资源整理与汇总（例如Logo），根据已有习惯养成建立本地文档，避免工作中与同事利益相冲突。

问题

从初次撰写到不断优化与否决

当文案一次又一次更改与否决时，对自己的输出产生怀疑，即使有了之前绘制原型图的文案撰写经验以及对产品的了解。

看到Word文档一次又一次红色与紫色的批注，关于具体文案的修改，后面自己尝试使用锤子便签的形式进行讨论，但效果几近慎微——同事还有其他的事情，没时间教你如何写文案。

此时考验的其实是自己的语文水平、理解水平了。

即使后来，他自己也会不断否决自己写过的文案，这样的操作使自己更加迷茫。

解决的方法

暂时发现多写多输出与讨论交流，暂无其他方法。

设计

本case中，自己曾受发布会样式所影响，初版评价正面，后续修改评价负面，修改页面样式时长占比较大。

被指幻灯片设计样式过于简单，后接受幻灯片参考，这些文件可作为留存。

参考网站：LinkedIn SlideShare、Pinterest、Behance等。

基础

  1. 衔接页面颜色若无动画，使用PDF浏览时需考虑（色块）连接性。

  2. 整体视觉元素的对其、居中与统一。

  3. 系统迭代后功能相应更改（如果有）

  4. 考虑使用B/C端效果图，展现形式更好-C端主要看效果图——考虑面向人群

  5. 非设计同事也会提出页面样式修改建议——仔细倾听，换位思考

5.1 做出方案给领导提供不同选择，让领导做选择题而非填空题。

核心页面

解决方案某种意义上说公司的产品，故核心页面需考虑：

  1. 封面：封面的呈现形式。考虑呈现形式，如果设计产品模型制作，则需节约时间，以及事先沟通继续保持。

  2. 公司介绍：关于数据可视化的展现方式进行多次确认，进行草稿留存，保留之前方案。

  3. 联系我们：联系我们页面如果使用大部灰色地图展现，则必须考虑政治相关：台湾、海南在地图中的必须出现，以及色调。

色调

本case中主色调使用Logo主色进行页面主色与点缀，若大面积出现灰色可能被指颜色沉重。

页面的色彩优化

部分空间需要进行优化时，可以列举页面总体颜色，之后使用相应颜色进行筛选与搭配，使得整页视觉观感协调。

插图

解决方案中根据产品功能进行绘制产品“近原型图”

寻找和内容一眼看上去相关的图，避免重找。

放置更加清晰的图片：手机界面，例如去掉手机框。

解决方案中的插图，包括：产品介绍、功能流程、背景图以及合作方Logo图，下面依次说明。

产品介绍图

根据受众人群，解决方案通常需要直接展示效果而非后台截图，此处我将它们进行结合。

示例数据

产品功能展示中需增加后台示例数据，便于前台展现，避免空白。

标注关键功能的形式

目的：标注出相应功能在页面的位置

  1. 放大镜形式：减弱其他
  2. 画圈形式：标注关键

功能流程图（如果有）

自己/内容运营供给

合作方Logo

是否有更新或删减，需第一时间核实。

考虑：放置之间间隙与协调，此处可参考各手机发布会直播平台界面。

动画

由于首先使用PDF格式进行宣发，动画可暂不考虑。

若需增加动画，考虑合适的动画效果，避免花哨。

修改

不同幻灯片有不同的调性。

“要命的”“优化”与”添加元素”

寻找是否有更好看的页面样式

不断进行删减优化，这话其实是一句无底洞。

无论是从内容口中还是领导口中甚至是设计口中——对于我来说这句话的出现，就意味着时间的无限延长。

制作过程中资源整理

与同事相协作，考虑到需要进行那些资源的收集与整理归档，可避免后续重复要资源。

空余时间及时进行聊天记录复盘与归档，可以弥补所需资源无法及时填充，导致发版时忘记。

以本次case为例，可供整理的资源包括文档、图片、音频与视频：

文档

不同文案文档word：产品文案、客户案例文案

图片-设计资源

内容与设计会给到不同类型的图片，都可以进行收集与整理。

资源需做到：确保视觉的统一，尽量保存所有色值与插图、合作方Logo

自己在这里是设计专业出身，所以这部分资源是自己进行部分插画设计的，就可以保留原文件。

版权

合作方Logo首选推荐使用公司市场部提供的，确保万无一失。

音频与视频

若有客户证言视频，可进行存档。

临时其他状况

临时事件

  1. 系统提示内存不足：及时进行保存与不同版本的先后备份
  2. 软件闪退：这一点无法预料，但可以根据系统内存不足的提示进行判断
  3. 同事时间延误：自己先把可以做的事先都做了，之后进行回顾与进度咨询
  4. 临时添加进来的事件，首先判断优先级，是否三分钟可以完成，若可以则优先去做，否则可以先记录一下。

关于草稿

自己在制作幻灯片还是绘制原型图时都有这样一点习惯，也在此处进行分享。

由于目前的原因，草稿仍不是很方便公开出来，但这是每一次进行构想的“试错与尝试”以及时间的保留。

如果可以，这里会放一个看片台的截图。如果硬要放，也必定全是马赛克，因为涉及利益相关。

协作前后沟通

工作中习惯养成

最后，分享一些自己在工作中养成的习惯，并和同事协作时选择提高效率的工具，给同事留下更好的协作体验。

  1. 成段发送文字与留言：可节省双方时间
  2. 考虑对方是否周末工作，防止该周任务堆积。
  3. 语雀进行在线预览幻灯片进度/Keynote/PPT协作功能提高效率
  4. 及时汇报目前需要提供的支持：考虑可能用到的支持：文案与图片等
  5. 文件压缩与下载时长，考虑直接浏览
  6. 内容运营等协作同事审核时间不确定

发问：如何绘制眼前一亮的解决方案

在文章《外包限制想象力》里我说过：

如果这么去做了，客户的口碑会更好，造成更好的效应呢？

也说不定会带来更多的客户，会有更多的外包项目。

这个case也逃不过，但这是我们自己的产品方案。

可能是在所有公司的环境下（不谈开放的公司），根据要求办事，你不能有自己的想法。

但相较于上一家，这个环境已经要好很多：可以将自己的想法在幻灯片中展现出来。

制作幻灯片不能“过于简单”，我们还是必须顺从受众人群（B端老板的审美）与“国内大众”的审美，即使他们都在套着各种各样的模板，别人一天这样做很多份PPT，而自己却需要使用一周的时间进行第一版，两周的时间进行删改。

从时间与效率上来说，就已经是超出预期了。

但这是自己的“内功”，与热爱原创的思路一直倔强的坚持。

其实，从样式上来说添加元素使得幻灯片内容不是那么过于简单，已经对于我来说是一个非常痛苦的事情：

不知要添加哪些元素，使用同一色系好还是使用“看起来”很花里胡哨的色系好（这当然不好）。

需要学习自己不能接受的制作幻灯片的形式，很难。但领导喜欢，看的B端老板喜欢，就成功了。

这毕竟不是做自己的内容，而是别人的。
